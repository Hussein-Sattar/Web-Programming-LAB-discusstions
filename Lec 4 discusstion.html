1- Not recommended to use tables for HTML layout, why?

ans//: Tables were originally intended for the display of tabular data, such as spreadsheets, and not for creating the overall layout of a web page. However, in the early days of web development, developers often used HTML tables to create page layouts because CSS support was limited, and it was challenging to achieve desired layouts without tables. Using tables for layout has several drawbacks:
Semantics, Accessibility, Maintenance and Flexibility, Performance
